
# 🚀 Telegram 频道强制订阅与防刷屏自动分发机器人

这是一个基于 **Cloudflare Workers** 架构开发的高性能、零成本 Telegram 自动回复机器人。专门用于**“订阅频道后才能获取节点/机密信息”**的业务场景。

本机器人完全通过 Cloudflare 的环境变量进行可视化日常维护，日常更新节点只需要在 CF 网页上修改纯文本即可，不需要接触任何代码。

### 🌟 核心特性
* 🔒 **死锁拦截级强力防绕过**：采用实时线上接口校验，用户关注后取消订阅或秒退频道，机器人下一秒立刻封锁，绝无白嫖可能。
* 🛡️ **一秒2次防刷熔断**：内置外层内存限流安检机，无论是已订阅用户还是黑客脚本恶意刷屏，超频消息就地抹杀，完美保护 Cloudflare 免费额度与 Telegram 账号安全。
* 📝 **纯文本变量驱动**：所有节点规则和关键词全部由 Cloudflare 变量 UI 驱动，像写记事本一样方便，**一行一个规则，用 `===` 分隔**。
* 🤝 **人性化客户引导**：内置小白式使用指南、点击自动复制代码块、并直达内置双向客服私聊机器人。

## ⚙️ 第二部分：要添加的环境变量明细

保存代码后，请前往 Worker 的 **`Settings` (设置) -> `Variables` (变量)** 页面，点击 **`Add`** 依次配置以下 **4个** 核心环境变量：

| 变量名称 (Key) | 作用描述 | 填写示例格式 |
| :--- | :--- | :--- |
| **`BOT_TOKEN`** | 你的 Telegram 自动回复机器人 Token | `（从@BotFather获取）` |
| **`CHANNEL_ID`** | 你的官方公网频道用户名（必须带 `@` 符号） | `@xxxx` |
| **`CHANNEL_LINK`** | 拦截用户时，引导跳转订阅频道的链接 | `https://t.me/xxxx` |
| **`NODE_RULES`** | 你的节点关键词纯文本数据库（一行一个规则） | *详见下方纯文本格式演示* |

### 📝 `NODE_RULES` 纯文本变量填写规范
在 `NODE_RULES` 的变量输入框中，直接像写记事本一样打字，格式严格保持：`关键词===回复内容`。
*支持通过 `\n` 进行回复内换行，节点链接前后请使用 \` 反引号包裹以在 Telegram 中实现点击自动复制代码块的效果。*

**直接复制并参考以下格式填入变量框：**
```text
香港节点===🇭🇰 **香港专线节点已下发**\n\n`vmess://香港节点链接11111111111`
日本节点===🇯🇵 **日本低延迟节点已下发**\n\n`vmess://日本节点链接22222222222`
备用线路===🔄 **防失联通用备用线路**\n\n`hy2://备用节点链接33333333333`

```

## 🚀 第三部分：保姆级零瑕疵部署教程

### 步骤 1：让 Bot 成为频道管理员（核心卡点）

1. 打开 Telegram，进入你的官方频道。
2. 进入频道设置 -> **管理员列表（Administrators）** -> 点击 **添加管理员**选项，将该机器人设置为管理员。保持默认权限开启并点击右下角 **`保存`**。（*注：机器人必须是频道管理员才能查询成员状态，否则系统会由于安全隔离机制默认全部拦截*）。

### 步骤 2：在 Cloudflare 创建并部署 Worker

1. 登录 [Cloudflare 控制台](https://dash.cloudflare.com/)。
2. 进入 **Workers & Pages** -> 点击 **Create Application** -> **Create Worker**。
3. 自定义 Worker 名称，点击最下方的 **Deploy（部署）**。
4. 部署成功后，点击页面上的 **Edit Code（编辑代码）** 按钮。
5. 从 **第一部分提供的超链接** 中复制完整的核心代码，全选粘贴进编辑器，点击右上角 **`Save and Deploy`（保存并部署）**。

### 步骤 3：绑定环境变量

1. 返回到该 Worker 的主控制面板，点击顶部导航栏的 **`Settings`（设置）**。
2. 选择左侧列表里的 **`Variables`（变量）**，在 **Environment Variables** 区域点击 **`Add`（添加）**。
3. 按照 **第二部分** 的表格规范，将 4 个变量完整填入，然后点击最下方的 **`Save and Deploy`（保存并部署变量）**。

### 步骤 4：激活 Webhook 通道（仅需执行一次）

请将以下链接中的 `<你的BOT_TOKEN>` 和 `<你的Worker独立域名>` 替换为你自己的真实数据，然后复制到浏览器中访问以激活通道：

```text
[https://api.telegram.org/bot](https://api.telegram.org/bot)<你的BOT_TOKEN>/setWebhook?url=https://<你的Worker独立域名>/

```

访问后，网页必须返回：`{"ok":true,"result":true,"description":"Webhook was set"}`（或者 `"Webhook is already set"`），至此机器人正式激活上线！

## 📱 第四部分：日常使用与维护方法说明

1. **未订阅用户体验**：向机器人发送任何指令或触发词，机器人都将**强行进行死锁拦截**，秒回警告卡片，同时下发前往频道订阅的内联按钮。
2. **已订阅用户体验**：发送 `/start` 会弹出格式完美的**使用指南**，以及直达你客服机器人的 **`✍️ 节点不能用？联系老板` (指向双向私聊机器人 @AGsykin_bot)** 按钮。发送预设的关键词（如 `日本节点`），将瞬间获得点击即可复制代码块的节点链接。
3. **日常维护与更新**：以后节点若失效或需添加新线路，**完全不需要再打开代码编辑器**。只需直接在 Cloudflare 网页上的 `NODE_RULES` 环境变量文本框里直接修改或回车增加新行，点击保存变量，1 秒内全网机器人数据实时同步生效。
