
# Telegram 智能分发与强制订阅引流系统 (高可用生产完满版)

这是一个基于 **Cloudflare Workers + 边缘内存沙盒 + Cloudflare KV** 架构的高性能、零成本、超低延迟 Telegram 自动回复引流系统。适用于“订阅指定公网频道后，才能通过私聊提取/解锁节点、私密网站、纯文字公告或海报图文”的私域闭环运营场景。

本系统在底层防灾、并发雪崩控制、Serverless 垃圾回收效率（GC）以及多端排版兼容性上进行了深度调优，去除了所有常驻定时器，实现纯净无状态的高并发处理。

---

## 💎 核心架构优势

* 🛡️ **双重网络防灾与 1 次自动重试**：底层 `telegramApi` 内置 `AbortController` 8 秒硬超时硬熔断机制。同时并入 1 次 500ms 延迟自动重试机制，可有效对冲 Telegram 网关突发的 502/504 临时故障或跨国骨干网网络震荡。
* 📉 **内存双轨 TTL 沙盒 (零 Timer 纯净态)**：引入抽象的 `TTLCache` 类统一接管 `userCache`（限流）、`menuCache`（菜单锁）和 `subCache`（订阅快照）。完全抛弃 `setTimeout` 与 `clearTimeout`，不对系统事件循环（Event Loop）留存任何微任务负担，允许 Serverless 容器极速热回收。
  * **订阅成功者**：内存锁定 **60 秒**，期间高频获取不同词条延迟全部为 **0 毫秒**。
  * **未订阅者**：仅缓存 **5 秒**，给真实用户留出点完关注折返回来发送词条的黄金真空期，发词即秒过。
* 🔒 **高并发 Promise 悬挂锁 (防雪崩)**：`getBotUsername` 函数并入 Promise 锁，并具备 24 小时自然老化机制。高并发入境时，后续并发请求直接排队挂载在同一个未决指针上，单容器生命周期仅发起一次真实 `getMe` 网络请求。
* ⚡ **一级常驻主哈希固化与脏数据清洗**：规则库由 $O(n)$ 线性扫描进化为常驻主 Map 的 $O(1)$ 瞬时咬合，黑名单采用 Set 哈希判定。数据入库和回退降维时并入严格的 `Array.isArray` 类型及非空字符串校验，物理级隔离一切因外部脏数据注入导致 `.toLowerCase()` 崩溃的隐患。
* 🧩 **关键词多空格压缩与井号穿透防护**：后台录入使用严格的正则非贪婪多行统包捕获，放行后面节点正文或备注里夹带的任何 `#` 特殊字符。同时全链路（包括用户发词匹配）自动通过 `\s+/g` 将多重连续空格压缩为单个标准空格，彻底消灭因格式不规范导致的识别分裂。

## 🚀 第二部分：高级进阶——分布式 KV 空间绑定

为了激活免后台动态管理控制台、选择性数字删除及动态黑名单功能，必须为 Worker 绑定 KV 命名空间。

### 1. 创建 KV 命名空间
1. 登录 Cloudflare 控制台，点击左侧导航栏的 **`KV`** 页面。
2. 点击右上角 **`Create a namespace`**，为其命名（推荐输入：`TG_BOT_DB`），点击 **`Add`**。

### 2. 将 KV 绑定至机器人 Worker
1. 进入机器人 Worker 主页面，点击 **`Settings` (设置)** -> 选择左侧的 **`Bindings` (绑定)**。
2. 在 **KV Namespace Bindings** 区域点击右侧的 **`Add Binding`**。
3. **严格按照以下命名规范填写**：
   * **变量名称 (Variable Name)**：必须手动输入全大写：**`TG_LIMIT_KV`**
   * **KV 命名空间 (KV Namespace)**：在下拉菜单中选中刚才创建的 `TG_BOT_DB`。
4. 点击最下方的 **`Save and Deploy` (保存并部署)**。

## 📡 第三部分：激活与 Webhook 打通

### 步骤 1：授权频道管理员
将机器人拉入您的官方公网分发频道中，并设置为管理员（保持默认的发布消息权限开启）。

### 步骤 2：注册 Webhook 通道（仅需执行一次）
将以下链接中的 `<你的BOT_TOKEN>` 和 `<你的Worker独立域名>` 替换为你的真实变量，在浏览器中敲回车访问。当页面返回 `{"ok":true,"description":"Webhook was set"}` 时，通道全面打通：
```text
[https://api.telegram.org/bot](https://api.telegram.org/bot)<你的BOT_TOKEN>/setWebhook?url=https://<你的Worker独立域名>/

```

### 步骤 3：动态身份菜单下发

部署完成后，管理员本人直接前往机器人私聊界面发送一次 `/start`。系统通过数字签名识别管理员身份后，会利用 `setMyCommands` 为管理员下发带有专属 `/manage` 后台的独立菜单。普通用户则只会下发基础的菜单，完全保持隐性隔离。

## 📱 第四部分：管理员后台精细控制使用指南

管理员在日常维护时，直接在机器人私聊框内发送 `/manage` 即可拉出最新的 KV 数据明细账单。

账单明细中的子备注升级为由 `<blockquote>` 标签包裹的缩进格式，在各端客户端（iOS、Android、Desktop）上均会自动渲染出一根灰色高雅的侧边竖线，排版极具对齐纵深感。

下方灰色代码块点击可直接一键复制，修改内容发送给机器人即可实时生效：

### 1️⃣ 添加/更新【文字、配置节点】

格式：`添加#关键词#频道显示的遮罩备注#用户私聊拿到的真实内容`

```text
添加#香港节点#香港专线#这里填写真实节点或任意回复内容

```

* **多行处理**：如果内容需要换行，直接在指令中输入 `\n\n` 即可，系统会在最终下发给用户时自动转换为标准的物理换行。
* **特殊字符处理**：得益于正则前置截断拦截，节点正文中夹带的任意多个 `#` 字符均能被完整保留，不会损坏节点资产。

### 2️⃣ 添加/更新【网页网址/群组绿色遮罩引流】

格式中的关键词直接使用完整的网址超链接：

```text
添加#[https://t.me/yourgroup#交流群#欢迎加入官方群组](https://t.me/yourgroup#交流群#欢迎加入官方群组)！

```

* **机制表现**：系统在频道广播时，长网址会被彻底隐形，并根据 `encodeURI` 规范完成单双引号（`%22` 与 `%27`）的深度安全替换，渲染为安全的蓝色超链接：[交流群]。用户一键点击即可无缝导流。

### 3️⃣ 添加/更新【海报图文大图引流】

1. 先把活动海报图片发在任意公开频道或群组，右键选择 **“复制消息链接 (Copy Post Link)”**，得到一个电报官方图片网络链接（如 `https://t.me/xqkin/123`）。
2. 将此链接作为网址参数输入指令：

```text
添加#[https://t.me/xqkin/123#查看海报#欢迎参加特惠活动](https://t.me/xqkin/123#查看海报#欢迎参加特惠活动)！

```

* **机制表现**：公网频道发出的文字超链接下方，会利用 Telegram 官方的链接预览机制，自动展开海报的预览大图，实现零内存消耗的图文广告引流。

### 4️⃣ 🔍 选择性精准数字删除与黑名单拦截

* **依据列表数字编号精准删除**：发送 `/manage` 呼出实时账单，看哪一条不顺眼（例如第 1 条），直接发送：
```text
删除#1

```


* **依据触发关键词匹配删除**：
```text
删除#香港节点

```


* **全局黑名单防线**：
* **拉黑恶意刷屏账户**：`拉黑#用户纯数字ID`（例如：`拉黑#6543210`），被封禁者发词会触发受限提示。
* **解除封禁**：`解黑#用户纯数字ID`（例如：`解黑#6543210`），恢复正常获取权限。

### 💡 运维防灾提示

* **防脏读自愈**：当您在 Cloudflare 控制台人肉删空了 `BLACKLIST_USERS` 或 `DYNAMIC_NODE_RULES` 时，Worker 会通过三元表达式瞬间将 `null` 转换为纯净空容器并重写常驻主哈希表。内存中不会有旧快照脏读，删除动作全局秒级同步生效。
* **刷新机制**：每次增删修改或拉黑后，聊天框内的旧消息不会变。只需**重新发送一次 `/manage**`，系统在 60 秒内存刷新窗口期（`CACHE_TTL.KV_REFRESH`）过后，会自动向 KV 提取最新数据，为您生成规整的盘点明细。
