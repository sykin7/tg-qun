
# Telegram 客服机器人 (Cloudflare Workers + D1 版)

一个基于 Cloudflare Workers 和 D1 数据库构建的高性能 Telegram 客服机器人。支持用户与客服群组的话题隔离会话、自动验证、反垃圾拦截、自动回复以及完善的 Webhook 后台配置系统。

## ✨ 核心功能特性

- **话题隔离会话**：自动为每位用户在客服群组创建独立话题，多客服并发接待不混乱。
- **双模式身份验证**：支持“点击按钮验证”和“4位验证码验证”，有效防止机器人轰炸。
- **漏斗式反垃圾机制**：集成本地词库与远程 GitHub 广告词库同步，支持大小写文本匹配与自动拉黑。
- **精细化消息过滤**：可按需开启/关闭纯文本、图片、视频、音频、贴纸、转发消息等各类型消息的转发权限。
- **智能自动回复**：支持基于正则表达式的关键词自动回复，拦截常见问题，减轻人工客服压力。
- **完善的权限体系**：区分主管理员（环境变量配置）与协管员（数据库配置），协管员可绕过验证直接对话。
- **全双向消息同步**：支持客服编辑消息后同步通知用户修改内容；支持用户编辑消息后同步通知客服。
- **安全备份机制**：支持配置独立的备份群组，所有用户消息均会留存副本。
- **可视化 Webhook 后台**：主管理员可直接在 Bot 私聊中通过 Inline Keyboard 完成所有系统配置，无需修改代码。

## 🛠️ 部署指南

### 1. 创建必要资源
1. 在 Cloudflare 控制台创建一个 **Workers** 项目。
2. 创建一个 **D1 数据库**（用于存储用户数据和配置）。
3. 在 D1 数据库的设置中，将其绑定到你的 Worker，绑定变量名必须设为：`TG_BOT_DB`。
4. 在 Telegram 中找 [@BotFather](https://t.me/BotFather) 创建一个 Bot，获取 `BOT_TOKEN`。

### 2. 配置环境变量
在 Cloudflare Workers 的设置 -> 变量中，添加以下环境变量：

| 变量名 | 必填 | 说明 |
| :--- | :---: | :--- |
| `BOT_TOKEN` | 是 | 从 BotFather 获取的 Bot Token。 |
| `ADMIN_GROUP_ID` | 是 | 客服管理群组的 ID（必须为超级群组且开启话题功能，Bot 需为群组管理员）。 |
| `ADMIN_IDS` | 是 | 主管理员的 Telegram User ID（多个用英文逗号 `,` 隔开）。 |
| `SPAM_URL` | 否 | 自定义远程垃圾词库的 URL（默认使用内置的 GitHub 广告词库）。 |

### 3. 部署代码与 Webhook 设置
1. 将本仓库代码粘贴到 Workers 的编辑器中并部署。
2. 首次部署后，代码会自动执行 `dbMigrate`，在 D1 数据库中自动创建所需的 `config`、`users`、`messages` 表及字段，**无需手动建表**。
3. 在浏览器访问 `https://api.telegram.org/bot<你的BOT_TOKEN>/setWebhook?url=<你的Worker域名>` 完成 Webhook 绑定。

## 📖 详细使用说明

### 一、 用户端交互指南 (普通用户视角)

1. **发起会话**：用户点击机器人链接或发送 `/start`。
2. **身份验证**：
   - **按钮模式**：机器人展示欢迎语，用户点击“✅ 点击这里验证身份”即可。
   - **验证码模式**：机器人展示欢迎语和一段4位验证码，用户需原样发送验证码通过验证。
3. **消息沟通**：验证通过后，用户发送的文本、媒体、文件等均会转发至客服群组对应的话题。
4. **消息编辑**：如果用户编辑了已发送的消息，系统会自动通知客服原内容与修改后的新内容。

---

### 二、 客服端操作指南 (管理员视角)

客服人员（即在 `ADMIN_IDS` 或协管员列表中的人员）通过 `ADMIN_GROUP_ID` 群组进行接待。

#### 1. 话题隔离机制
每当新用户首次发消息，Bot 会在客服群组自动创建一个以 `用户名 | UserID` 命名的话题。之后该用户的所有消息都在此话题内，互不干扰。

#### 2. 回复用户
客服只需在对应的话题内**直接发送消息**（支持文本、图片、语音、视频、文件、贴纸等），Bot 会自动将其转发给该话题绑定的用户。
- **修改消息**：客服在群组中编辑自己发送的回复，Bot 会向用户发送一条系统通知，展示原内容与修改后的新内容。

#### 3. 用户资料卡与管理操作
在用户的话题内，顶部会有一张“👤 用户资料卡”消息，底部带有三个管理按钮：
- **🚫 屏蔽此人 / ✅ 解除屏蔽**：拉黑/解封用户。被拉黑的用户发送的消息将被机器人静默丢弃。
- **🔕 静音通知 / 🔔 解除静音**：屏蔽该用户后续消息的群组提醒（不影响转发，只是群组不响铃）。
- **📌 置顶此消息**：将该资料卡置顶在当前话题，方便后续查找。重新置顶会更新数据库中记录的资料卡 ID。

### 三、 主管理员后台配置手册 (核心)

**仅限 `ADMIN_IDS` 中的主管理员**，在私聊机器人发送 `/start` 时，会唤出**机器人主配置菜单**。

#### 菜单 1：📝 基础配置 (验证模式)
用于设置用户首次访问的验证流程。

- **🔄 切换验证模式**：在“点击按钮验证”和“4位验证码验证”之间切换。
- **📝 编辑欢迎/提示语**：
  - 点击后，直接发送纯文本即可设置。
  - **格式说明**：支持换行、支持 Emoji。由于安全转义机制，**不支持 HTML 标签**（如 `<b>` 会原样显示）。
  - **文本示例**：
    ```text
    👋 欢迎来到我们的客服中心！
    为了给您提供更高效的服务，请先简要描述您遇到的问题。
    ⏰ 人工客服在线时间：周一至周五 9:00 - 18:00
    ```

#### 菜单 2：🤖 自动回复管理
设置常见问题自动回复。**注意：一旦触发自动回复，消息将不会转发给人工客服。**

- **➕ 新增自动回复规则**：
  - **格式要求**：必须使用 `三个英文等号 (===)` 分隔关键词和回复内容。
  - **格式模板**：`关键词表达式===回复内容`
  - **匹配机制**：使用正则表达式进行模糊匹配。
  - **示例 1（包含匹配）**：`价格===您好，产品价格请查看：xxx.com`
  - **示例 2（多词匹配）**：`你好|您好|在吗===您好！客服小助手为您服务，请问有什么可以帮您？`
  - **示例 3（精确匹配）**：`^菜单$===功能菜单：\n1. 查询订单\n2. 售后服务`
- **🗑️ 管理/删除现有规则**：点击后进入列表，点击对应按钮可逐条删除。

#### 菜单 3：🚫 关键词屏蔽管理
反垃圾核心模块，拦截广告与违规信息。

- **➕ 新增屏蔽关键词**：
  - 直接发送文本。匹配机制为**不区分大小写的纯文本包含匹配**（例如设置“代开”，则“代开发票”也会被拦截）。
- **✏️ 修改屏蔽次数阈值**：
  - 设定一个数字（默认为 5）。用户触发屏蔽词达到该次数后，将被机器人**自动永久拉黑**。
- **🔄 同步远程 Spam 词库**：
  - 一键从配置的 `SPAM_URL`（默认 GitHub）拉取大量广告屏蔽词并合并到本地词库。
- **🗑️ 管理/删除现有关键词**：进入列表可逐条删除。

#### 菜单 4：🔗 按类型过滤管理
精细控制允许转发到客服群的消息类型，点击按钮即可切换“允许/屏蔽”状态。

- **转发来源控制**：允许/屏蔽来自其他用户/群组/频道的转发消息。
- **媒体类型控制**：允许/屏蔽音频/语音、贴纸/GIF、图片/视频/文件。
- **基础内容控制**：允许/屏蔽包含链接的消息、允许/屏蔽纯文本消息。
- **安全兜底机制**：如果媒体转发被关闭，对于联系人、位置、投票、骰子等特殊未列明消息，系统将自动拦截，防止绕过。

#### 菜单 5：🧑‍💻 协管员授权设置
- **主管理员**：由环境变量 `ADMIN_IDS` 设定，拥有最高权限。
- **协管员**：主管理员可在此添加协管员 ID 列表（多个用逗号隔开）。
- **权限说明**：协管员可以**绕过验证**直接与机器人对话，并**有权在客服群内回复用户**，但无权打开主配置菜单。

#### 菜单 6：💾 备份群组设置
- 可设置一个额外的“备份群组 ID”。设置后，所有用户发给机器人的消息，都会复制一份发送到该备份群组，用于聊天记录留存审计。

### 四、 系统自动化日志机制

机器人内置了完善的日志追踪体系，全自动运行，无需手动干预：

1. **📋 用户资料卡汇总**：
   机器人会在客服群内自动创建此系统话题。每当新用户发消息，系统会在此话题发布一条该用户的资料卡，并附带“跳转到会话窗口”的快捷按钮，方便客服总览所有咨询用户。
2. **🚫 屏蔽与静音名单**：
   系统自动创建此话题。当用户被屏蔽或静音时，系统会在此话题更新其状态卡片。
3. **状态全局同步**：
   如果客服在用户的话题内拉黑了用户，系统会自动同步更新“汇总话题”和“屏蔽名单话题”中该用户的按钮状态，确保所有地方的资料卡状态绝对一致。

## 💡 常见问题与操作提示

1. **如何取消正在输入的配置？**
   在任何要求发送文本的配置环节（如输入欢迎语、新增规则时），随时发送 `/cancel` 或点击界面下方的“❌ 取消”按钮即可中止操作并返回上一级。
2. **为什么有的用户发了消息客服群没反应？**
   请检查以下可能：
   - 用户未通过验证。
   - 用户被拉黑了。
   - 用户发送的消息类型在“按类型过滤管理”中被屏蔽了。
   - 用户发送的消息触发了“关键词屏蔽”。
   - 用户发送的消息触发了“自动回复”（触发后不转发给人工）。
3. **修改了配置后数据库卡顿没刷新？**
   点击列表按钮删除规则时，受限于 Telegram API，界面可能不会立即刷新。点击“⬅️ 返回”后重新进入列表即可看到最新状态。
